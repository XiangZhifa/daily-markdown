## 1. 产品概述与目标用户

### 1.1 产品概述
一款**本地部署、多账户隔离**的 Markdown 知识管理工具。

用户在注册并登录后，可快速编写、组织、检索和统计自己的 Markdown 文档。

产品以「快」和「私密」为核心体验，所有文档基于用户身份严格隔离，兼顾个人知识管理与多成员共用环境下的数据安全。

### 1.2 产品目标
- 提供**毫秒级响应**的文档增删改查体验。
- 通过标签体系实现灵活的多维度分类。
- 让用户能按日期、名称、标签快速定位任何一篇文档。
- 自动生成按月维度、按标签的统计视图，辅助用户回顾写作习惯。
- 通过用户系统实现**一人一空间**，同一设备多人使用互不干扰。

### 1.3 目标用户
- **个人技术写作者**：需要管理大量技术笔记、博客草稿，并希望数据完全私有。
- **知识管理爱好者**：喜欢用标签搭建个人知识体系，定期复盘写作量。
- **家庭或小团队共享设备用户**：多人共用一台电脑，各自拥有独立的文档库。
- **轻量项目文档维护者**：小团队内用 Markdown 记录会议、周报，但需要按成员隔离视图。

## 2. 详细功能列表

### 2.1 用户管理（新增）
| 功能编号 | 功能名称 | 功能描述                                                   |
|----------|----------|--------------------------------------------------------|
| U-01 | 用户注册 | 用户通过用户名 + 密码创建账号。密码需满足复杂度要求（≥8位，含字母和数字）。注册成功后自动跳转文档主页。 |
| U-02 | 用户登录 | 使用已注册的邮箱/用户名和密码登录。支持“记住我”功能（7天免登录）。                    |
| U-03 | 用户登出 | 清除当前会话，返回登录页。                                          |
| U-04 | 会话持久化 | 基于 JWT Token 实现。前端存储 Token，请求时自动携带。Token 过期后自动跳转登录页。   |
| U-05 | 个人信息查看与修改 | 可查看用户名/邮箱、注册时间；支持修改密码（需验证原密码）。                         |
| U-06 | 账户注销 | 用户可申请注销账号，系统将异步清理该用户所有文档、标签及统计数据，不可恢复。需二次密码确认。         |

### 2.2 文档管理（CRUD）
| 功能编号 | 功能名称 | 功能描述 |
|----------|----------|----------|
| F-01 | 创建文档 | 一键新建空白 Markdown 文件，自动生成默认标题（如“未命名-日期”），并进入编辑状态。文档自动关联当前登录用户。 |
| F-02 | 编辑文档 | 支持分屏/实时预览的 Markdown 编辑器，提供常用工具栏（加粗、斜体、标题、列表、代码块、插入图片等）与快捷键。 |
| F-03 | 保存文档 | 手动保存(Ctrl+S)与自动保存（编辑停止2秒后触发）。保存时自动提取标题、更新修改时间。 |
| F-04 | 删除文档 | 软删除（移入回收站，仅限本用户），支持从回收站恢复或彻底删除。 |
| F-05 | 文档列表 | 以表格/卡片形式展示当前用户文档，可自定义显示列（标题、创建时间、修改时间、标签）。支持多选批量删除/批量添加标签。 |
| F-06 | 文档预览 | 在列表内支持快速预览摘要（前3行内容），无需打开编辑器。 |

### 2.3 标签管理
| F-07 | 添加标签 | 在编辑器内为当前用户文档添加一个或多个标签，输入时有已有标签自动补全（仅限本用户标签）。 |
| F-08 | 移除标签 | 从文档中移除指定标签（支持批量操作）。 |
| F-09 | 标签重命名 | 重命名标签后，该用户所有关联文档同步更新。 |
| F-10 | 标签删除 | 删除标签时解除该用户所有文档的该标签关联。 |
| F-11 | 标签列表 | 侧边栏展示当前用户所有标签及其文章数量，支持点击过滤文档。 |

### 2.4 搜索与过滤
| F-12 | 按标题搜索 | 输入关键词实时模糊搜索当前用户的文档标题。 |
| F-13 | 按日期过滤 | 提供日期区间选择器或快捷选项（今天、最近7天、本月、自定义范围），基于文档的创建时间或修改时间过滤。 |
| F-14 | 按标签过滤 | 支持单选标签过滤，也可多选标签（AND/OR 逻辑可切换，默认 OR）。 |
| F-15 | 组合搜索 | 支持「日期 + 标题 + 标签」三者任意组合查询，结果实时更新（限定当前用户数据）。 |

### 2.5 统计看板
| F-16 | 月度文章统计 | 以日历热力图或柱状图展示当前用户每月创作文章数量。 |
| F-17 | 标签分类统计 | 饼图或条形图展示当月（或选中月）当前用户各标签文章数量与占比。 |
| F-18 | 统计时间切换 | 用户可切换查看不同月份/年份的统计数据。 |
| F-19 | 点击下钻 | 在统计图表中点击某标签或某一天，自动跳转到对应的文档列表过滤视图。 |

### 2.6 数据与设置（后续版本建议）
| F-20 | 导入 Markdown 文件 | 支持批量导入本地 .md 文件，自动解析标题并作为新文档入库（归属当前用户）。 |
| F-21 | 导出文档 | 支持导出为 Markdown、HTML、PDF 格式。 |
| F-22 | 多主题 | 编辑器支持浅色/深色主题切换，并提供几种代码高亮风格。 |
| F-23 | 回收站管理 | 回收站界面，支持查看已删文档、恢复、清空。 |
| F-24 | 自动备份 | 定时自动备份 SQLite 数据库到本地指定目录。 |

## 4. 界面设计要求

### 4.1 整体布局（桌面端优先）
应用分为**未登录态**与**登录态**两大界面体系。

**未登录态**：
- **登录/注册页**：居中卡片式设计，左侧品牌区（产品 Logo + 简介），右侧表单区。支持切换登录/注册，密码重置链接（后续版本）。
- 注册表单：用户名（或邮箱）、密码、确认密码、提交按钮。
- 登录表单：用户名/邮箱、密码、“记住我”复选框、提交按钮。

**登录态**（主工作区）：
- 沿用经典**三栏布局**：
    - **左侧边栏（窄，可折叠）**：顶部显示用户头像/用户名，下方标签列表 + 底部统计入口，最底部设置与登出入口。
    - **中间文档列表（可调宽度）**：文档卡片/表格，顶部搜索栏与过滤条件。
    - **右侧编辑/预览区**：上部分编辑器（支持分屏），下部分状态栏。

### 4.2 页面详情

**1) 登录/注册页**
- 背景简洁，卡片居中。
- 输入框实时校验（邮箱格式、密码强度、两次密码一致）。
- 切换链接：“已有账号？立即登录” / “没有账号？立即注册”。
- 登录错误提示（账号或密码错误，账号锁定等）。

**2) 文档列表页（主视图）**
- 顶部：搜索框（支持标题模糊搜索）、日期范围选择器、标签多选下拉。
- 顶部右侧可显示用户头像，点击弹出下拉菜单（个人设置、退出登录）。
- 中间：文档列表，每项展示标题、标签徽章、修改时间、摘要预览（前80字符）。
- 底部：分页或无限滚动。
- 左侧“+”按钮新建文档。

**3) 编辑器视图**
- 左侧文档列表变为当前文档的目录大纲（自动解析标题层级），或隐藏。
- 右侧分为：工具栏、编辑区（代码视图）与预览区（左右分屏/切换）。
- 标签编辑区：在标题下方有标签输入框，支持添加/删除标签。
- 自动保存状态提示（“已保存/保存中…”）。

**4) 统计看板**
- 顶部月份切换器（箭头切换月份/年份）。
- 左侧柱状图（每日文章数），右侧饼图（标签分布）。
- 图表支持鼠标悬停显示详情，点击可跳转过滤列表。

### 4.3 设计原则
- **品牌信任感**：注册/登录页需体现专业与安全感（锁图标等，即使本地部署也建议美化）。
- **状态清晰**：登录过期或未登录时，所有 API 请求重定向至登录页，全局拦截。
- **简洁克制**：减少视觉噪音，聚焦内容。
- **响应式**：虽然主打桌面端，但需保证在笔记本（1366×768）下可用，最小侧边栏可折叠。
- **明暗主题**：浅色主题(背景色：#FFFFFF)、深色主题(背景色：#1A1A1A)，默认为 深色主题，通过Css变量实现切换。
- **一致性**：所有图标、间距、字号遵循统一设计规范。

## 6. 代码风格与架构模式

### 6.1 代码风格
- **TypeScript 强制**，开启严格模式。
- 使用 ESLint + Prettier 统一代码格式，配置 Vue 3 和 NestJS 官方推荐规则。
- 提交规范：遵循 Conventional Commits（feat/fix/docs 等），配合 Husky + lint-staged。
- 命名规范：
    - 前端组件 PascalCase，文件名 kebab-case。
    - 后端服务/控制器按功能划分，使用有意义的名词。
    - 数据库表/列名使用 snake_case。

### 6.2 前端架构模式
- 采用**功能模块化**：`documents`、`tags`、`statistics` 模块独立。
- 组件分层：
    - **通用组件**（按钮、标签输入框、日期选择器）
    - **业务组件**（文档卡片、标签徽章、统计图表）
    - **页面组件**（DocumentList.vue, EditorView.vue, StatisticsView.vue）
- 状态管理：每个模块内部使用 Pinia store，API 请求与缓存逻辑封装在 store action 中。
- 编辑器与预览通过 props/events 通信，保持解耦。
- 增加 **auth 模块**：Pinia store 管理用户信息与 Token。路由守卫 `beforeEach` 检查 Token 有效性，无 Token 或过期跳转登录页。
- **请求拦截器**：axios 自动附加 `Authorization: Bearer <token>`，响应拦截器捕获 401 状态码执行登出跳转。

### 6.3 后端架构模式
- NestJS 模块结构：
    - `documents` 模块（CRUD + 搜索）
    - `tags` 模块（标签 CRUD，关联管理）
    - `statistics` 模块（聚合查询，按月/标签统计）
- 使用 **Repository 模式** 封装数据库操作，便于单元测试。
- DTO 验证：使用 class-validator 确保输入安全。
- 统一异常处理与响应格式 `{ code, data, message }`。
- `auth` 模块：
    - `auth.controller`：处理登录、注册、登出、个人信息接口。
    - `auth.service`：实现验证逻辑、JWT 签发、密码哈希。
    - `local.strategy`：用户名密码验证。
    - `jwt.strategy`：JWT 负载解析，注入请求用户信息。
- 所有业务模块（documents, tags, statistics）的服务层方法均接收 `userId` 参数，并在查询时添加 `where: { userId }` 条件，实现数据隔离。
- 使用全局 Guard (`AuthGuard`) 保护所有需要登录的接口，公共接口（注册/登录）使用自定义装饰器 `@Public()` 放行。

## 7. 非功能性需求

### 7.1 性能
- 文档列表首屏加载时间 < 300ms（本地环境）。
- 编辑器打开 10万字符文档时，输入延迟 < 50ms。
- 搜索与过滤响应时间 < 100ms（数据库索引优化）。
- 统计查询利用 SQL 聚合，避免全量数据传输，月度统计加载 < 500ms。
- 登录接口响应时间 < 200ms，注册 < 300ms。
- Token 校验采用轻量 JWT 解析，不引入 Redis（本地单用户场景），性能损耗可忽略。
- 数据隔离查询需确保 `userId` 索引有效，避免全表扫描。数据库在 `documents.userId`, `tags.userId` 上建立索引。

### 7.2 安全（强化）
- 密码使用 bcrypt/argon2 哈希存储，不可逆。
- JWT 密钥存储在后端环境变量，前端不暴露。Token 有效期设为 24 小时，Refresh Token 策略（V2.1 引入）。
- 注册/登录接口需做速率限制（如单 IP 每 60 秒最多 5 次尝试），防止暴力破解（可用 `@nestjs/throttler`）。
- 所有用户输入做 XSS 防注入和参数校验。
- 数据隔离强制在后端实现，禁止前端通过任何参数切换 userId。
- 用户注销后，该用户关联数据必须物理删除（或通过异步任务安全清理）。
- 本地部署场景下，建议引导用户设置数据库文件所在目录的访问权限。
- 前端渲染预览时，对 HTML 做 XSS 清洗（DOMPurify）。
- 后端所有输入做参数校验，避免 SQL 注入（TypeORM 已做参数化）。
- 禁止执行用户提交的脚本内容。

### 7.3 可靠性与数据完整性
- 自动保存需保证内容不丢失，使用防抖和异常捕获。
- 文档删除使用软删除，支持恢复。
- 数据库操作使用事务，如批量添加标签、删除标签同步解除关联，保证数据一致性。
- 注册时邮箱/用户名唯一性校验（数据库唯一约束 + 后端逻辑校验）。
- 用户注销操作放入事务，确保文档、标签、统计相关数据一并移除。
- 自动保存等功能与用户绑定，保存时确保 userId 不可篡改。

### 7.4 可用性
- 纯键盘操作支持：常见编辑快捷键、Ctrl+S 保存、Ctrl+F 聚焦搜索等。
- 错误提示友好，网络或数据库错误时给出可理解的信息。
- 支持国际化预留（至少支持中文/英文，当前版本只实现中文）。
- 登录表单支持回车提交。
- 密码错误或账号不存在时给出明确但不过度暴露信息的提示（如“账号或密码错误”，而非“账号不存在”）。
- 注册时密码强度实时指示器。

### 7.5 可维护性
- 前端项目采用组件测试（Vitest + Vue Test Utils），后端单元测试（Jest）。
- 关键 API 编写 e2e 测试。
- 清晰的目录结构与模块职责，代码注释必要处。
- Auth 模块独立，可单独测试。
- 编写集成测试覆盖注册、登录、Token 刷新等流程。

## 8. 限制条件与边界场景

### 8.1 限制条件
- **单用户登录**：同一账号不限制多设备登录（后续版本可加入互斥策略）。
- **用户数量**：SQLite 限制下，用户数建议不超过 100 人，避免并发写入冲突。
- **用户名规则**：用户名必须为 3-20 位字母、数字或下划线，不区分大小写但存储统一小写；邮箱格式符合 RFC 标准。
- **密码规则**：至少 8 位，包含字母和数字，支持特殊字符。
- **会话**：仅使用 Access Token，有效期 24 小时；刷新机制留至后续版本。
- **不支持并发协同编辑**：无 WebSocket，同一时间一个编辑器实例。
- **数据库并发限制**：SQLite 写入串行，频繁自动保存与统计查询可能出现短暂阻塞，通过 WAL 模式优化。
- **最大单文档尺寸**：建议限制 5MB 文本（约 500 万字符），超出时提示用户拆分。
- **标签数量上限**：单个文档最多关联 20 个标签，系统总标签数无硬性限制但建议不超过 5000。
- **图片处理**：编辑器内的图片采用本地相对路径或 base64 小图（限制 1MB），不提供图床功能。
- **全文搜索**：当前版本仅支持标题模糊搜索，不对文档正文做全文索引；(后续版本 可选集成 SQLite FTS5 实现正文搜索。)

### 8.2 边界场景
- **注册冲突**：
    - 用户名已存在：提示“用户名已被注册”。
    - 邮箱已存在：提示“该邮箱已被注册”。
    - 密码不一致：实时校验，提交时再次验证。
- **未登录访问**：直接输入主界面 URL 时，重定向至登录页；登录成功后回跳至目标页面。
- **Token 过期**：发起请求时若返回 401，前端自动登出，并提示“会话已过期，请重新登录”。
- **用户注销中登录**：若后台清理未完成时用户再次登录，应提示“账号正在处理中，请稍后”。
- **用户删除自己的唯一文档**：无特殊处理，回收站按用户隔离，清空回收站时彻底删除。
- **多个用户同时写入 SQLite**：SQLite 单写锁，高并发时可能造成排队，通过 WAL 模式缓解。在一般个人使用场景下可接受。
- **忘记密码**：当前版本不提供自助重置，后续版本通过邮箱验证实现。
- **数据隔离测试**：必须确保 A 用户无法通过猜测 ID 访问 B 用户的文档。后端需以 `userId` 作为强制过滤条件。
- **空状态**：
    - 无任何文档时，文档列表展示引导页“创建第一篇文档”。
    - 无标签时，标签侧边栏显示“添加标签”引导。
    - 统计无数据时显示“本月暂无文章”。
- **超大数据量**：文档数量超过 1 万篇时，列表默认分页加载（每页 50 条），搜索需优化索引；虚拟滚动（后续版本）可考虑。
- **标签重复与重名**：系统不允许创建同名标签（忽略大小写）。
- **删除标签牵连**：删除标签时，需提示影响文档数量，并允许用户取消。
- **数据库损坏**：启动时检测 SQLite 完整性，提示恢复或重建。
- **自动保存冲突**：用户正在编辑时，若另一个地方（如外部工具）修改了数据库文件，下次保存可能覆盖。建议提示“文件已被外部修改”或锁定机制。
- **文件路径中特殊字符**：数据库存储路径时做好 Unicode 处理，文件系统操作验证合法性。
